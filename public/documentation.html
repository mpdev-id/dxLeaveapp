API Documentation

This documentation outlines the API endpoints and usage for different user roles within the dxLeaveApp system.

Role: Member (Employee)

Endpoints available for standard employees.

POST /api/register

Description: Register a new user account.

POST /api/login

Description: Login to get an authentication token.

GET /api/user

Description: Fetch own user profile.

POST /api/leave-requests

Description: Submit a new leave request. The request starts in 'Draft' status.

GET /api/leave-requests

Description: View own leave request history.

PATCH /api/leave-requests/{leaveRequest}

Description: Update a leave request that is in 'Draft' status. To submit for approval, set current_status to 'Pending'.

Example Body (to submit):

{
    "leave_type_id": 1,
    "start_date": "2025-12-20",
    "end_date": "2025-12-22",
    "reason": "Holiday vacation",
    "current_status": "Pending"
}

POST /api/logout

Description: Logout and invalidate the token.

Role: Approver (SPV, SL, ASMEN, TL, Manager)

Approvers have all Member permissions, plus the ability to approve or reject leave requests.

GET /api/leave-requests

Description: View pending leave requests that require their approval.

PATCH /api/leave-requests/{leaveRequest}/approve

Description: Approve or reject a leave request.

Request Body:

{
    "action": "Approved",
    "comments": "Enjoy your vacation!"
}

Set action to "Rejected" to reject the request.

Role: Super Admin

The Super Admin has full access to the system for managing master data.

All endpoints are prefixed with /api/admin/master

GET /users

Description: Get a list of all users.

POST /users

Description: Create a new user.

GET /users/{user}

Description: Get details of a specific user.

PUT/PATCH /users/{user}

Description: Update a user's information.

DELETE /users/{user}

Description: Delete a user.

GET /departments

Description: Full CRUD for managing departments.

GET /leave-types

Description: Full CRUD for managing leave types.

GET /public-holidays

Description: Full CRUD for managing public holidays.

GET /employee-entitlements

Description: Full CRUD for managing employee leave entitlements.

GET /leave-requests

Description: Full CRUD for managing all leave requests.
